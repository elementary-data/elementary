Cloud tests run in Elementary Cloud, outside your dbt pipeline. They don't require configuration in your dbt project — Elementary activates them automatically once your data warehouse is connected.

There are three Cloud tests:

| Test | What it detects |
|---|---|
| [Cloud freshness test](/cloud/features/anomaly-detection/automated-freshness) | Tables that haven't been updated within their expected window |
| [Cloud volume test](/cloud/features/anomaly-detection/automated-volume) | Unexpected drops or spikes in row count |
| [Data contract test](/cloud/features/data-tests/data-contract-test) | Schema mismatches against a defined contract — column names, types, nullability |

Cloud tests query warehouse metadata only — not the data itself — so they add no compute cost and run continuously without pipeline dependencies.

## Configuration

Cloud tests are activated automatically on your sources, but you can add, edit, or remove them at any time — from the UI or via the [Elementary AI Agent](/cloud/ai-agents/test-recommendation-agent).

Each test can be configured to match how your data actually behaves:
- Adjust sensitivity and thresholds to reduce noise
- Set a custom training period so the model learns from the right history
- Use SLA-based mode for freshness when you need a fixed update window

See [configuration options](/cloud/features/anomaly-detection/monitors-configuration) and [feedback](/cloud/features/anomaly-detection/monitors-feedback) for details.

## Metrics

You can also collect and monitor custom metrics on your data using the [`elementary.collect_metrics`](/cloud/features/anomaly-detection/metrics) dbt test. Metrics are synced to Elementary Cloud and can have anomaly detection applied automatically.

## Test coverage

Use the [test coverage screen](/cloud/features/data-tests/test-coverage-screen) to see which assets have coverage, identify gaps, and add tests directly from the UI.
