In **Elementary Cloud**, pipeline task performance is monitored natively. Elementary automatically creates a [Cloud pipeline task performance](/cloud/features/anomaly-detection/cloud-pipeline-task-performance) monitor for every model, seed, and snapshot, with anomaly detection or a static SLA. No dbt tests required.

<Note>
  Looking for the dbt-test method (defining tests on top of the Elementary artifacts schema)? That approach is for Elementary OSS — see [Performance alerts (OSS)](/oss/guides/performance-alerts).
</Note>
