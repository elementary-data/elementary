## What you can send alerts about

- Model run failures
- Failures and/or warnings of dbt tests (including Elementary dbt package)
- Failures and/or warnings Elementary Cloud test tests
    - Automated volume tests
    - Automated freshness tests
- dbt source freshness failures

## Enriching alerts

You can enrich your alerts by adding properties to tests and models in your **`.yml`** files. The supported attributes are: 

- owner
- tags
- subscribers
- description

## Configuring Microsoft Teams alert rules

Elementary Cloud allows you to create rules that route your alerts. Each rule is a combination of a filter and a destination.

The `Microsoft Teams` webhook you choose when connecting your `Microsoft Teams` integration is automatically added as a default alert rule, that sends all the alerts to that webhook without any filtering. To modify, deactivate or add more rules, simply navigate to the “Alert routing rules” page in the left menu.
