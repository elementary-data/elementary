- **New selector filters for alerts**
  - We added 2 new filters to `edr monitor`, `resource_type` and `status`!
  - This means it is now possible to run `edr monitor` separately for warnings and failures, and send them to different slack channels!
  - Thanks [Josh Elston-Green](https://elementary-community.slack.com/team/U04CSH84TBN) for requesting this, and also [Pascal Moreau](https://elementary-community.slack.com/team/U04GR7G5G0K) for opening the [issue on Github](https://github.com/elementary-data/elementary/issues/630)
  - See the [docs](https://docs.elementary-data.com/guides/alerts-configuration#filter-alerts) for more details
- **Test your alerts in "dev" mode**
  - You can now run alerts that ignore your custom channels and suppression settings. This is intended for testing alerts in dev that you don't want everyone to panic about. Just try `edr monitor --override-dbt-project-config`
  - Thanks [Snir Israeli](https://elementary-community.slack.com/team/U04HZGN1AAY) for opening this feature request!
